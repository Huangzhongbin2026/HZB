# 核心测试用例

## 1. 任务管理
1. 创建已下单加急任务：必填字段完整，返回 taskId。
2. 创建未下单评估任务：orderNo 为空，成功入库。
3. 创建客期变更任务：优先级边界值 1 和 5 均通过。
4. 任务转派：目标处理人为空时报错 code=1001。
5. 任务关闭：关闭原因为空时报错 code=1001。
6. 列表筛选：按任务类型+状态组合筛选结果正确。

## 2. 配置管理
1. 保存请假配置：开始时间小于结束时间。
2. 请假代理生效：命中产品型号后处理人替换为代理人。
3. 消息开关关闭后：对应事件不触发推送。

## 3. 报表统计
1. Dashboard 汇总值与任务主表一致。
2. 准时率计算公式正确：done_on_time / done_total * 100。
3. 超时任务统计仅包含未在 SLA 内完成任务。

## 4. 消息推送
1. 任务创建触发推送成功后记录 t_msg_push_log。
2. 推送失败进入重试队列，重试次数递增。
3. 超过最大重试次数进入死信队列。

## 5. 鉴权与安全
1. 缺失 Authorization 返回 code=1002。
2. 非授权角色访问配置接口返回 code=1003。
3. 审计日志记录创建/转派/关闭/配置变更。

## 6. 性能与稳定性
1. 任务列表接口 500 并发下 P95 < 300ms。
2. 报表接口缓存命中率 >= 85%。
3. MQ 堆积告警阈值触发后 5 分钟内恢复。
