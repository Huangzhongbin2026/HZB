# 辅助功能模块表结构说明

## 1. 索引优化
- 请假状态查询：`idx_leave_user_time` 支持人员+时间范围快速筛查。
- 虚拟产品匹配：`uk_virtual_model` 保证完全匹配查找 O(logN)。
- 区域匹配：`idx_area_precise` 优先命中精准规则，关键词规则走 `idx_area_keyword`。

## 2. 约束规则
- 请假配置与统筹用户强关联：`sys_aux_leave_config.user_id -> sys_user.id`。
- 代理管理原统筹/代理统筹均强关联用户表。
- 虚拟产品型号唯一。
- 消息路由 `route_code` 唯一。
- 代理产品 `product_model + original_user_id` 唯一。

## 3. 匹配逻辑说明
- 请假匹配：按当前日期命中请假区间。
- 代理匹配：请假命中后按 `product_model + original_user_id` 找代理。
- 区域匹配：精准命中优先，未命中再按关键词规则。

## 4. 性能说明
- 虚拟产品匹配结果写入 Redis，建议 TTL 10 分钟。
- 高频匹配接口响应目标 <= 100ms。
