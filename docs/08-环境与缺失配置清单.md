# 环境与缺失配置清单

## 1. GitHub 代码托管
- 必填：仓库地址（例如 git@github.com:<org>/<repo>.git）。
- 必填：团队协作权限（Owner/Maintainer/Developer）。
- 建议：开启分支保护（main 仅 PR 合并，禁止直接 push）。
- 安全：请使用 Personal Access Token（PAT）或 SSH Key，不要使用账号密码。

## 2. 数据库与缓存
- 二选一：
  - 方案A：本地容器数据库（已提供 docker-compose.local.yml）。
  - 方案B：你提供测试/生产数据库地址，我来替换配置。
- 若走方案B，需要你提供：
  - MySQL：host、port、db、username、password
  - Redis：host、port、password（如有）
  - 网络策略：白名单或内网访问方式

## 3. 后端运行环境
- JDK 17（必需）
- Maven 3.9+（当前机器缺失）
- 启动参数：spring.profiles.active=dev/prod

## 4. 前端运行环境
- Node.js >= 20
- pnpm = 8

## 5. CI/CD（建议尽快补齐）
- GitHub Actions 或 GitLab CI 发布流水线。
- 自动化步骤：lint -> test -> build -> deploy。
- 版本打标：发布分支自动生成 tag。

## 6. 监控与日志（生产必配）
- 日志中心：ELK/Loki
- 指标监控：Prometheus + Grafana
- 告警通道：企业IM/邮件

## 7. 配置中心（后续建议）
- 当前为 application-*.yml 模板。
- 建议上线前切换为 Nacos/Consul 并管理敏感配置。
