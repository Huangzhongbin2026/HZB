# 系统管理模块开发手册（技术视角）

## 1. 接口前缀
- /api/v1/supply/system/

## 2. 权限校验
- 使用 @RequirePermission("sys:menu:add") 进行接口级校验。
- 页面按钮权限通过 v-permission 指令控制。
- 字段权限通过 FieldPermissionFilter 工具处理。

## 3. 日志埋点
- 核心操作接口添加 @SysLog 注解。
- 切面自动记录模块、类型、结果、耗时、IP。

## 4. 数据权限
- 在查询方法上使用 @DataScope(tableAlias="t", ownerField="owner_user_id", deptField="dept_code")。
- 拦截器解析用户角色并拼接 SQL 条件。

## 5. 扩展接口预留
- /users/sync/feishu 用户同步。
- /roles/permissions/batch 角色权限批量配置。
- /logs/push/third-party 日志第三方推送。

## 6. 常见排查
- 菜单不显示: 检查 sys_role_menu 绑定与权限缓存是否刷新。
- 字段不生效: 检查 sys_role_field_permission 合并优先级。
- 日志丢失: 检查 @SysLog 注解与异步执行线程池。
