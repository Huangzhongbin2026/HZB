# UI 规范

本规范基于当前产品已经形成的暖中性色背景、浅表面层、蓝色强调色和类 Obsidian 的信息架构整理，目标是让 Web、桌面端、移动端在不牺牲平台特性的前提下保持同一视觉语言。

## 1. 设计目标

- 简约：减少装饰性噪声，优先用层级、留白、对齐表达结构。
- 美观：保留纸感暖色底和轻玻璃表面，但避免过强拟物。
- 一致：三端共享颜色、文字层级、圆角和间距体系。
- 平台适配：移动端保留更大触控区、更少并列信息和更强主次。

## 2. 核心视觉 Token

### 2.1 颜色

- 页面背景：`#f4f1ea` 到 `#ebe7df` 的暖灰渐变。
- 背景氛围色：低透明度蓝色与赭色径向光斑，只做气氛，不承载信息。
- 卡片表面：`rgba(255,255,255,0.9)`。
- 次级表面：`rgba(248,246,241,0.92)`。
- 边框：`rgba(79, 68, 57, 0.12)`。
- 主文字：`#2d2a26`。
- 次文字：`#756a5f`。
- 强调色：`#4f6bff`。
- 强调色深阶：`#3550c8`。
- 强调色浅阶：`rgba(79,107,255,0.12)`。
- 危险色：`#d45454`。
- 成功色：`#23825a`。

### 2.2 圆角

- 大卡片：20px 或移动端 28rpx。
- 中控件：16px 或移动端 22rpx。
- 小按钮/标签：12px 或移动端 18rpx。
- 胶囊标签：999px。

### 2.3 阴影

- 卡片主阴影：`0 18px 40px rgba(37, 32, 28, 0.08)`。
- 辅助阴影：`0 10px 24px rgba(37, 32, 28, 0.05)`。
- 移动端固定底栏阴影可以略强，但不超过 `0 18rpx 42rpx rgba(104, 84, 62, 0.12)`。

## 3. 字体与层级

### 3.1 Web / 桌面端

- 一级标题：24px，700。
- 二级标题：18px 到 20px，700。
- 卡片标题：16px 到 18px，700。
- 正文：14px 到 15px，行高 1.7。
- 辅助文字：12px，行高 1.6。
- Eyebrow：11px，全大写，字距 0.16em。

### 3.2 移动端

- 页面标题：48rpx 到 50rpx，700。
- 分区标题：32rpx 到 34rpx，700。
- 正文：26rpx 到 28rpx，行高 1.7。
- 辅助文字：24rpx，行高 1.6。
- Eyebrow：22rpx，全大写，字距 4rpx。

## 4. 间距体系

### 4.1 Web / 桌面端

- 页面外边距：14px 到 16px。
- 卡片内边距：16px。
- 紧凑面板：14px。
- 元素间距：4 / 8 / 12 / 16 / 24。
- 三栏布局默认列距：14px 到 16px。

### 4.2 移动端

- 页面左右边距：24rpx。
- 区块间距：20rpx 到 24rpx。
- 卡片内边距：24rpx 到 26rpx。
- 组件垂直节奏：12 / 16 / 20rpx。
- 固定底部栏与内容区保留至少 160rpx 安全空间。

## 5. 控件规范

### 5.1 按钮

- 主按钮：深色或品牌蓝渐变底，白字，只用于主要动作。
- 次按钮：浅底 + 细边框，用于非破坏性次级动作。
- 图标按钮：只保留 1 个动作语义，桌面端 32px 到 40px，移动端 76rpx 到 82rpx。
- 危险按钮：只用于删除、清空、断开等不可逆动作。

### 5.2 输入框

- 输入面高度桌面端不低于 34px，移动端不低于 86rpx。
- 焦点态统一使用强调色边框 + 4px 或等价 rpx 外发光。
- 文本域优先纵向拉伸，不使用固定死高度的长文本输入框。

### 5.3 标签与状态

- 标签、筛选 chip、状态 pill 统一使用浅底、细边框或淡强调底。
- 同一行内不混用超过 3 种视觉强度。

## 6. 布局规则

### 6.1 Web / 桌面端

- 延续 rail + explorer + main + inspector 的四区结构。
- 主编辑区必须是视觉焦点，左右面板不应比主区更重。
- inspector 以分段或 tab 切换呈现，不直接堆砌长列表。

### 6.2 移动端

- 采用单列内容流，不做桌面式三栏压缩。
- 底部导航只保留一级入口，图标按钮最小触控面 76rpx。
- 命令面板、设置、选择器使用 action sheet、picker、modal 等系统友好模式。
- 笔记详情中的属性、链接、标签采用分组卡片，而不是侧边栏思维硬搬。

## 7. 交互规范

- Hover 只作为增强，不作为唯一区分方式。
- 焦点态、激活态、选中态必须都能只靠颜色和背景区分。
- 自动保存、同步状态、AI 处理中等系统反馈必须用统一的辅助文字或状态 pill 展示。
- 动画时间以 180ms 到 220ms 为主，避免拖沓。

## 8. 本次检查结论

### 8.1 Web

- 已具备较完整的视觉 token 基础。
- 调整方向：统一与桌面端的强调色、文字色、边框色和圆角标尺。

### 8.2 桌面端

- 已形成和 Web 接近的工作台结构。
- 调整方向：继续去掉局部尺寸漂移，保持面板、按钮、卡片统一圆角和统一留白。

### 8.3 移动端

- 视觉方向正确，但局部页面存在重复写死色值与尺寸。
- 调整方向：抽成全局移动 token，保持更大触控区、更高信息块间距和更轻的视觉密度。

## 9. 执行要求

- 新页面优先复用 token，不允许重新发明新色板。
- 没有明确业务优先级时，先保证排版、间距和状态清晰，再加装饰。
- 三端新增组件提交前至少自查：文字层级、边距、圆角、边框透明度、焦点态、激活态、空状态。
