# 辅助功能模块架构与流程

## 1. 融合方案
- 辅助功能模块归属系统管理服务 `supply-system-service` 下的子域。
- 对外统一接口前缀：`/api/v1/supply/system/aux/`。
- 五类子模块：请假配置、虚拟产品、消息推送、请假代理产品型号、区域统筹划分。
- 主要功能模块通过“匹配接口”引用辅助数据，避免重复维护。

## 2. 数据关联流程图

### 2.1 请假配置 -> 代理产品 -> 主任务匹配
```mermaid
flowchart LR
    A[主任务传入: 统筹人+产品型号+日期] --> B[查询请假配置]
    B --> C{统筹人是否请假}
    C -- 否 --> D[保持原统筹]
    C -- 是 --> E[查询请假代理产品型号]
    E --> F{命中 产品型号+原统筹}
    F -- 是 --> G[替换为代理统筹]
    F -- 否 --> H[进入默认统筹池]
```

### 2.2 虚拟产品匹配流程
```mermaid
flowchart LR
    A[主任务传入产品型号] --> B[Redis缓存查询]
    B --> C{命中缓存}
    C -- 是 --> D[返回自动回复内容]
    C -- 否 --> E[查询虚拟产品表]
    E --> F[完全匹配产品型号]
    F --> G[写入Redis缓存]
    G --> D
```

### 2.3 区域统筹匹配流程
```mermaid
flowchart LR
    A[输入: 销售部门+省份+部门关键词+项目关键词] --> B[精准匹配 销售部门+省份]
    B --> C{是否命中}
    C -- 是 --> D[按关键词规则细分匹配]
    C -- 否 --> E[关键词匹配策略]
    D --> F[返回区域统筹]
    E --> F
```

## 4. 关键规则
- 请假时间同一人员不允许重叠。
- 请假代理产品：`product_model + original_user_id` 唯一。
- 虚拟产品型号完全匹配，不做模糊替代。
- 区域统筹匹配优先级：精准 > 关键词。
- 所有导入导出、开关变更、匹配调用均可记录系统日志。
