# 客期变更数据传输接口文档

## 1. 接口说明
- 接口名称：客期变更任务数据传输
- 适用场景：用户端“客期变更任务”在第三步确认后，按所选产品逐条写入任务管理平台“客期变更任务管理”
- URL：`/api/v1/supply/task/delivery-change/submit`
- Method：`POST`
- Content-Type：`application/json`

## 2. 请求头
| 名称 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| sysId | string | 是 | 全局函数字段 |
| sign-server-auth | string | 是 | 签名接口返回的 signature |

## 3. 请求体
```json
{
  "mode": "CRM",
  "requesterName": "张某某",
  "contractNo": "HT20260322001",
  "taskType": "客期提前",
  "changeReason": "项目现场要求提前到货",
  "allowPartialShipment": "同意",
  "delayReason": "",
  "delayProof": "",
  "repeatFlag": "是",
  "regionPlanner": "华南区域统筹",
  "selectedData": [
    {
      "市场代码名称": "MK001",
      "产品型号": "RG-S7808C",
      "物料描述": "核心交换机",
      "变更数量": 2,
      "客户期望交期": "2026-06-20",
      "约定交期-ODC": "2026-06-28",
      "客期提前至": "2026-06-22",
      "客期延后至": "",
      "计划可出货日期": "2026-06-24"
    }
  ]
}
```

## 4. 字段映射规则
- `审批编号`：系统生成，格式 `KQBGyyyyMMddHHmmssX`
- `任务类型`：第一步选择“客期提前/客期延后”
- `是否重复提交`：重复校验命中后，用户确认继续提交时为“是”
- `统筹评估回复`：默认空，管理页支持失焦自动保存
- `状态`：默认“待评估”；当统筹评估回复填写后自动转“已评估”并记录完成时间
- `代理区域统筹`：若区域统筹休假则按规则匹配代理人
- `代理产品统筹`：按产品型号匹配对应产品统筹

## 5. 校验规则（核心）
- 合同编号不能为空
- 客期提前：目标日期必须满足 `>= 今天` 且 `< 最大客期`
- 客期延后：目标日期必须满足 `> 今天` 且 `> 最小客期`
- 第二步仅允许勾选“市场代码名称非空”的行
- 至少勾选一条数据后才允许进入第三步
- 第三步提交前需重新获取签名，签名有效期为 5 分钟

## 6. 响应体
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "approvalNo": "KQBG202603221045301",
    "count": 2
  },
  "traceId": "..."
}
```

## 7. 错误码（核心）
- `2201`：合同编号不能为空
- `2202`：客期变更类型不合法
- `2203`：未选择可提交的产品行
- `2204`：客期变更日期不满足规则
- `2205`：请求头缺少 `sysId` 或 `sign-server-auth`
- `2206`：signature 已过期，请重新获取
