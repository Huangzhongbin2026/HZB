# 未下单咨询数据传输接口文档

## 1. 接口说明
- 接口名称：未下单咨询任务数据传输
- 适用场景：用户端“未下单咨询任务”提交后，按产品型号逐条写入任务管理平台“未下单交期评估任务”
- URL：`/api/v1/supply/task/unordered-consult/submit`
- Method：`POST`
- Content-Type：`application/json`

## 2. 请求头
| 名称 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| sysId | string | 是 | 全局函数字段 |
| sign-server-auth | string | 是 | 签名接口返回的 signature |

## 3. 请求体
```json
{
  "mode": "CRM",
  "requesterName": "张某某",
  "crmNumber": "20251014000222",
  "needTime": "2026-04-30",
  "remark": "10月14号报备，项目进入招标阶段",
  "taskType": "重要项目",
  "repeatFlag": "是",
  "selectedData": [
    {
      "CRM编号": "20251014000222",
      "产品型号": "MM850",
      "产品名称": "某系列",
      "数量": 4,
      "要求发货日期": "2026-04-30",
      "是否备货": "是",
      "勾选或取消备货时间": "2025-10-14",
      "项目盘点详情url": "https://marketplan.com.cn/userid=20251014000222",
      "一级部门": "某某部门",
      "三级部门": "重庆区域",
      "机会所有人": "桂某某",
      "项目名称": "某某项目",
      "客户名称": "某某客户",
      "是否退市报备": "否",
      "产品流速": "100"
    }
  ]
}
```

## 4. 字段映射规则
- `交期咨询任务编号`：系统生成，格式 `WXDyyyyMMddHHmmssX`
- `备注`：用户第一步填写的备注
- `需求日期`：用户第一步填写（同时转换毫秒时间戳存储）
- `产品统筹`：根据产品型号调用产品统筹接口匹配
- `任务类型`：历史重复弹窗中“普通项目/重要项目”选择结果
- `是否重复提交`：历史重复弹窗选择“再次提交”时为“是”
- 其余字段：来自 CRM 计划接口或产品型号输入解析结果

## 5. 响应体
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskNo": "WXD202603220945301",
    "count": 2
  },
  "traceId": "..."
}
```

## 6. 错误码（核心）
- `2109`：没有可提交的数据，请至少选择一条记录
- `2110`：全部产品型号无法查询到系统数据及格式不合法
- `2111`：部分产品型号无法查询到系统数据及格式不合法（返回具体型号）
- `2113`：请求头缺少 `sysId` 或 `sign-server-auth`
- `2115`：signature 已过期，请重新获取
