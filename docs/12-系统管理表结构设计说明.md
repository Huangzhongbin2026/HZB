# 系统管理表结构设计说明

## 1. 字段级权限存储方式
- sys_role_field_permission 以 role_id + biz_table + field_code 为唯一键存储权限。
- permission_type 取值: VISIBLE、EDITABLE、HIDDEN。
- 合并规则: 多角色叠加时按优先级 EDITABLE > VISIBLE > HIDDEN。

## 2. 多角色权限叠加逻辑
- 菜单/按钮权限: 角色权限并集。
- 数据范围权限: 以最大范围优先，规则 ALL > CUSTOM_DEPT > DEPT > SELF。
- 字段权限: 同字段按优先级合并。
- 角色继承: inherit_role_id 链路递归展开后再合并权限。

## 3. 日志存储与检索优化
- 独立表 sys_operation_log，避免影响业务表。
- 复合索引: (oper_module, oper_time)、(oper_user, oper_time)。
- 内容检索采用 FULLTEXT（或ES旁路索引）。
- 日志清理按时间分段删除，默认保留>=365天。

## 4. 高频查询建议
- 菜单树: parent_id + sort_no 索引。
- 用户登录: account 唯一索引。
- 飞书ID校验: feishu_id 唯一索引。
- 字典模糊检索: item_name 普通索引 + 前缀匹配策略。

## 5. 多环境同步
- sys_menu 增加 env_tag 字段。
- 提供菜单导入导出 JSON 接口，用于 dev/sit/prod 配置同步。
